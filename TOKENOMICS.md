## 📋 概述

Solana AI City 将采用 **双代币经济模型**：

| 代币 | 符号 | 类型 | 用途 |
|------|------|------|------|
| **$CITY** | CITY | 实用代币 | 游戏内所有交易 |
| **$GOV** | GOV | 治理代币 | 协议治理、长期价值 |

### 经济平衡公式

```
收入 = 建筑产出 + 战斗奖励 + 任务奖励

支出 = 升级消耗 + NFT购买 + 游戏消费 + 手续费

平衡条件:
收入 ≈ 支出 (±5% 浮动)

如果不平衡:
- 收入 > 支出 → 增加消耗渠道
- 收入 < 支出 → 增加收入来源
```

## 🔒 质押机制

### 质押池

```
┌─────────────────────────────────────────────────────────┐
│                   质押池结构                             │
├─────────────────────────────────────────────────────────┤
│                                                          │
│  基础池 (任意数量)                                       │
│  ├── APY: 15%                                          │
│  ├── 锁定期: 无                                         │
│  └── 最低: 1 GOV                                      │
│                                                          │
│  投票池 (需投票)                                         │
│  ├── APY: +5% (总计 20%)                               │
│  ├── 锁定期: 7 天                                       │
│  └── 要求: 每季至少投 1 票                              │
│                                                          │
│  长期池 (1年锁定)                                        │
│  ├── APY: +10% (总计 25%)                              │
│  ├── 锁定期: 365 天                                     │
│  └── 解锁后: 可继续锁定 +10%                            │
│                                                          │
│  最大 APY: 30%                                          │
│  (基础 15% + 投票 5% + 长期 10%)                        │
│                                                          │
└─────────────────────────────────────────────────────────┘
```

### 质押奖励计算

```
奖励 = 质押量 × (基础APY + 投票加成 + 长期加成) × 时间

示例:
质押 1,000 GOV
├── 基础: 1,000 × 15% = 150 GOV/年
├── 投票: +50 GOV (5%)
└── 长期: +100 GOV (10%)
                    ─────────
总计:              300 GOV/年 (30%)
```
